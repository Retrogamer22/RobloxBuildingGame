# BuildBlox — Repository Guide

This repo holds **two versions** of BuildBlox, kept separate on purpose:

- **`dev/`** — where new features and big updates are built. This is ahead of live.
- **`public/`** — a snapshot matching the **live** public game. Quick bug fixes happen here.

Shared stuff (`Assets/`, `docs/`, `TetxureWebsite/`) lives at the top level and is the same for both.

---

## Why two folders?

`dev` contains unreleased work (e.g. portal/Nether system, text blocks, fire handler) that must **not** leak into the live game before it's ready. Keeping `public` as its own folder lets you patch the live game without dragging in half-finished dev features.

The rule of thumb:
- A fix made in **public** should also be carried into **dev** (so it isn't lost at the next release).
- Work in **dev** does **not** go to public until release day.

---

## The two Argon projects

Each folder has its own Argon project on its own port:

| Folder | Project name | Port | Studio place |
|--------|-------------|------|--------------|
| `public/` | BuildBlox-Public | 8000 | the live public experience |
| `dev/` | BuildBlox-Dev | 8001 | the dev/beta experience |

To work on one version:
1. In **VS Code**, open that folder (`File → Open Folder` → `dev` or `public`).
2. Run **Argon: Serve** (Ctrl+Shift+P → Argon → Serve).
3. In **Studio**, open the matching place and click **Connect**.

### ⚠️ The one rule that prevents disasters
**Only run ONE Argon session at a time, and confirm which port/place you're connected to before accepting any changes.** Two sessions fighting, or syncing the wrong direction, is how things break. If a sync looks like it's going the wrong way (pushing your files UP to a place you didn't mean to overwrite), **disconnect immediately.**

### If Argon won't populate empty folders
Argon does **not** auto-create the `src/` subfolders. If a fresh pull from Studio fails with `os error 3 / cannot find the path specified`, the subfolders (`Shared`, `Server`, `Client`, etc.) must exist on disk first. Create them empty, then re-serve.

## Everyday workflow (GitHub Desktop)

You don't need the terminal for normal work — GitHub Desktop handles commits and backup.

**To save a checkpoint (any time you've done work worth keeping):**
1. Open **GitHub Desktop**.
2. It shows every changed file across *both* `dev/` and `public/` in the left panel.
3. Type a short **Summary** (e.g. "Fix vine hitbox lingering after break").
4. Click **Commit to master**.
5. Click **Push origin** (top bar) to send it up to GitHub.

That's the whole loop. The repo covers both folders, so one commit captures whatever you changed, wherever you changed it.

> GitHub Desktop does **not** auto-commit — and that's intentional. *You* decide when work is "done enough" to snapshot. Don't let Argon's constant syncing become commit noise.

---

## Release day (dev → live)

When a dev update is ready to go live:
1. In **Studio**, open the **dev** place.
2. Use **Publish to Roblox As...** to overwrite the **public** place with the dev one.
3. Back on disk, bring `dev/src/` into `public/src/` so the files match what you just published (copy the changed files over, or use git to sync), then commit in GitHub Desktop.

This keeps `public/` honest with what's actually live, so the next time you open the public place, Argon doesn't try to push stale code back over it.

---

## Carrying a public bug fix into dev

Since `dev` and `public` are folders (not branches), the simplest way:
- Make the same edit in `dev/src/`, **or**
- Copy the fixed file from `public/src/` over to the matching path in `dev/src/`.

Then commit in GitHub Desktop. (If this becomes frequent, a branch-based setup can automate it — ask about it then.)

## What's ignored by git

`.gitignore` keeps these out of the repo:
- `*.rbxl` / `*.rbxlx` — binary place files (published via Studio, not tracked)
- `sourcemap.json` — auto-generated by Argon
- editor/OS junk (`.vscode/`, `.DS_Store`, logs)

---

## Quick reference

- **Backup is on GitHub** (private repo). Commit + push via GitHub Desktop.
- **Dev work:** `dev/` folder, port 8001, dev place.
- **Public fixes:** `public/` folder, port 8000, live place.
- **One Argon session at a time. Check the direction.**
